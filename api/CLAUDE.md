# api/ — 로컬 백엔드 (FastAPI)

- 이 앱은 **AWS Lambda + API Gateway의 로컬 스탠드인**이다(로컬 우선 결정). 여기서 정의한 **엔드포인트 계약을 React 프론트가 그대로 소비**하고, 배포 시 Lambda 핸들러로 옮긴다 — 그래서 계약을 함부로 바꾸지 않는다.
- **현재값은 매 요청마다 실시간 수집**한다(캐시 미경유, 원칙1).
- `GET /api/macro/indicators` → `{indicators: {key: IndicatorPoint|null}, partial_failure: [key]}`. `IndicatorPoint`의 `as_of(date)`는 FastAPI 인코더가 ISO 문자열로 직렬화한다.
- `GET /api/macro/regime` → `{...judgement, indicators_used, partial_failure}`. `judgement`는 `macro.engine.judge_regime`의 반환 계약(2축: regime/recommended_cash_ratio/confidence/axes/key_drivers/params/vix_panic/missing_indicators/raw_data — 구 `votes`·`override`는 `axes`·`vix_panic`으로 대체·폐기). `axes`는 `{cycle:{score,sign}, sentiment:{score,sign}}`, `key_drivers` 원소는 `(label, axis, direction)` tuple, `vix_panic`은 `vix>35` 표시 플래그(블랭킷 오버라이드 아님). 흐름: `collect_macro_indicators`로 실시간 수집(캐시 미경유) → 국면 4지표(`t10y2y→yield_spread`/`hy_spread`/`vix`/`fear_greed`)만 `.value` 추출해 엔진 입력 dict 구성(None·부분실패·키부재는 제외, 임의 기본값 금지) → `judge_regime` 호출. `indicators_used`=엔진에 실제 넣은 `{엔진키: value}`, `partial_failure`=국면 4지표 중 못 쓴 엔진키. `dollar_index·gdp`는 수집돼도 판정 제외(매핑은 `_REGIME_INPUT_MAP` 단일 출처). `key_drivers`의 tuple 은 JSON 배열로 직렬화된다. 판정은 전부 규칙 코드(LLM 미개입). **`indicator_breakdown`**(추가): 판정근거 카드용 `[{key,label,value,unit,zone,axis,source,thresholds:{lo,hi}}]`(누락 지표도 value/zone=None) — `macro.engine.regime_breakdown(engine_input)` 이 생성(엔진 SSOT 분류). 수치 표면화이지 판정 아님.
- `GET /api/macro/regime/history?months=` → **국면 이동 궤적(족적)** `{months, interval:"quarterly|semiannual|annual", step_months, points:[{date, cycle_score, sentiment_score, regime, recommended_cash_ratio, vix_panic, missing_indicators}], available, partial_failure, note?}`. FRED 3지표 + 공포탐욕 월 히스토리를 **병렬 수집**(`infra.parallel.fetch_parallel`) → `macro.regime_history.build_trajectory` 가 월별 `judge_regime` 재현(**판정은 코드·결정적**, LLM 미개입) → **범위별 표본 간격 다운샘플**(`trajectory_step`+`downsample_trajectory`: 1년=분기·2년=반기·3년=연, 정상 12/24/36개월=4/4/3점, 최근 앵커·판정 불변·표시 밀도만). 응답 `interval`(표본 코드, 구 `"monthly"` 폐기)+`step_months` 추가. `months` clamp **1..60**(기본 **36**). **진행 중 당월은 결정적 제외**(`_current_month_kst()` → build_trajectory `exclude_month`; FRED 부분월 평균이 '현재' 점·캐시가 되는 것 방지). 지표별 수집 실패는 그 지표만 제외(공포탐욕 결측이어도 심리축 VIX 로 판정 → 궤적 유지)·`partial_failure` 에 기록. **확정 과거값이라 캐시**(`macro:history:regime:{months}`, TTL 1일·표본화도 캐시 반영; **available 만 저장** — 불가/전결측/정합실패[수집 성공이나 어느 달도 양축 미충족]는 미저장). **항상 200 graceful**. 상세는 `macro/CLAUDE.md`.
- `GET /api/macro/indicators/{key}/history?months=` → `{key,label,unit,source,thresholds,months, points:[{date,value}], available, note?}`. 국면 4지표 1개의 **월단위 히스토리(기본 1년)** — 카드 클릭 시. FRED 3지표(yield_spread/hy_spread/vix→VIXCLS)는 월 다운샘플, `fear_greed`는 CNN graphdata best-effort. **캐시**(`macro:history:`, TTL 1일; `available`일 때만 저장·불가/실패 미저장). 불가·실패는 `available:false`+note(**항상 200 graceful**), 판정 4지표 아닌 key 는 400. `indicator_meta`(engine)가 key 검증·메타 SSOT.
- `POST /api/chat` → `{text, popups:[{name,args}]}`. body `{session_id, message}`. 서버 세션(`chat.session.get_session`)에서 히스토리 유지. judgement는 `api/main.py::live_judgement()`로 실시간 계산 — `macro_regime`과 **동일 헬퍼 단일 출처**(캐시 미경유). 판정·팝업 결정은 `chat.chat`(LLM은 설명만).
- `POST /api/chat/stream` → **SSE**(`text/event-stream`, `StreamingResponse`). 이벤트를 `data: {json}\n\n`로 프레이밍, `ensure_ascii=False`. 이벤트 `{type:"stage"|"token"|"popups"|"done"}`, stage enum `analyze|regime|generate|summarize`. 헤더 `Cache-Control:no-cache`+`X-Accel-Buffering:no`(프록시 버퍼링 방지). **guardrail 경로는 `live_judgement` 미실행**(진입점 classify로 먼저 걸러 FRED 낭비 0). 기존 `POST /api/chat`(논스트림)은 그대로 유지 — 스트리밍 미지원 환경 폴백. 프론트는 fetch 스트림 리더로 소비(POST+body라 EventSource 불가).
- **워치리스트(§3 모듈 3)** — `GET /api/watchlist?sort_by=` → `{items, regime, sort_by, partial_failure}`(enriched: 라이브 시세 + **매수/매도 목표가 상태** + 스파크라인, registered 순). item 에 매수(`target_price`/`target_status`/`distance_to_target`)+**매도(`sell_target_price`/`sell_target_status`/`sell_distance_to_target`)** 키. 국면별 종목 진입신호(`entry_signal`/`regime_gate`)는 폐기(항목3), `regime` 블록은 국면명만. `POST {ticker, stock_name?, reason?, target_price?, sell_target_price?}`(추가/upsert; **신규**가 `WATCHLIST_MAX_ITEMS`=30 도달이면 **409**; 미제공 매수/매도 목표가는 기존값 보존), `DELETE /{ticker}`, `PATCH /{ticker} {target_price?, sell_target_price?}`(**바디에 온 필드만** 부분 갱신 — `model_fields_set`→`store.update_targets` sentinel, 매도만 보내면 매수 불변·null=해제; 미등록 404·음수 422). **매매 주문 API 없음**(목표가 저장·조회만). 라우트는 `api.detail`의 `_build_kis_client`·`_build_judgement`를 재사용. 저장 계약은 `watchlist/CLAUDE.md`.
- **선택형 차트(일봉/주봉 × 기간)** — `GET /api/detail/{ticker}/chart?period=D|W&range=3m|1y|3y|10y` → `{ticker, period, range, candles:[...], stage_segments:[{stage,start_date,end_date}], current_stage, partial_failure}`. period/range **화이트리스트 폴백**(불량→D/1y). `_CHART_RANGE_DAYS`(3m92/1y366/3y1096/10y3653)로 날짜창 매핑 → `chart.fetch_chart_series`(KIS ~100/콜 **페이지네이션**, collectors/CLAUDE) → 캔들 + `stage_segments_for_chart`(표시 시계열로 리본 **재계산**). **일봉 무캐시**(원칙1·오늘봉 형성) · 불량 ticker 400 · KIS 실패 **항상 200 graceful**(빈 candles+partial_failure). ⚠ **정량 요약(RSI/MA/현재 대순환 단계·GrandCyclePanel)은 번들[일봉]에 pin** — 이 라우트는 표시 데이터만(차트 탐색이 판정을 안 바꿈). 번들 `_fetch_chart`(요약용 6개월 일봉·단일 호출)는 불변. `_resolve_client` 옵션 인증(본인/공유 키).
- **[P2] 구조화 리포트(§6.5b)** — `POST /api/detail/{ticker}/report` → LLM이 `StockReport`(Pydantic) JSON 생성·검증(실패 1회 재시도→폴백 정량요약+`validation_failed`) 후 히스토리 저장. `GET /api/detail/{ticker}/report/history`. 상세는 `chat/CLAUDE.md`.
- **현재 보는 화면 컨텍스트** — `POST /api/chat/context {session_id, kind, args}`(`api/chat.py`): 사용자가 보고 있는 패널(balance/watchlist/stock_report)을 세션 핀 컨텍스트로 고정 → 이후 챗이 그 데이터를 근거로 답변. **화면 데이터는 프론트가 보내지 않음** — 서버가 `chat.view_context.build_view_context`로 kind/args 재조회(환각·조작 차단). 비데이터 kind(macro_dashboard·manage_watchlist)·조회 불가·kind 없음은 이전 핀 해제. **항상 200**(백그라운드 핀은 게이트키핑 아님). 상세는 `chat/CLAUDE.md`.
- **애널리스트 리포트(네이버 수집·요약·상담 연계)** — `api/reports.py`(기존 RAG reindex/status 라우터 확장): `POST /api/reports/fetch?limit=N`(1~50 클램프) → `analyst_service.fetch_and_summarize`(수집→다운로드→요약→저장 병렬), 항상 200 + `{fetched,new,skipped,failed}`(수집/요약 실패 graceful). `POST /api/detail/{ticker}/analyst-reports/fetch?limit=N`(1~30 클램프) → `analyst_service.fetch_and_summarize_for_ticker`(**그 종목만** itemCode 필터로 수집→요약→저장), 항상 200 + 카운트(graceful·`assert_valid_ticker` 400). ⚠ 종목 상세의 "가져오기"는 이 종목별 라우트를 써야 한다 — `/api/reports/fetch`(전체 최신 피드)를 쓰면 대부분 종목의 섹션이 비어 보인다(전 종목 섞인 피드에 그 종목이 없어서). `GET /api/detail/{ticker}/analyst-reports` → `{ticker, reports:[...]}`(저장 요약 최신순, 없으면 `[]`; `assert_valid_ticker` 400). `POST /api/chat/report-context`(`api/chat.py`) `{session_id, ticker, report_id}` → 저장 요약을 세션 상담 컨텍스트로 핀(요약 본문은 서버가 store 조회 — 프론트 신뢰전송 없음, 환각 차단; ticker/report_id 없으면 해제, 없는 리포트 404). 요약·자문은 '리포트 내용 인용'(에이전트 판정 아님)·면책. 상세는 `chat/CLAUDE.md`·`collectors/CLAUDE.md`.
- **시황(매크로) 챗 상담 + 금일의 요약** — `POST /api/chat/market-outlook-context`(`api/chat.py`) `{session_id, report_id}`(시황은 시장 전체라 **ticker 없음**) → 저장 시황 요약을 **애널리스트와 같은 세션 핀 슬롯**(`report_context`)에 핀(서버가 `market_outlook_store.get(report_id)` 조회 — 프론트 신뢰전송 없음·환각 차단; report_id 없으면 해제·없는 리포트 404). `POST /api/macro/market-outlook/summary`(`api/macro_outlook.py`) → 최근 **5개** 시황 종합 **10줄 '금일의 요약'**(`market_outlook_combined.summarize_recent_outlooks`·저장 요약만·PDF 재다운로드 0·`graceful_counts` 폴백·항상 200). 요약/종합=리포트 인용(시장 판정 아님)·면책. 상세 `chat/CLAUDE.md`.
- **사이트 통계(헤드라인)** — `GET /api/stats`(`api/stats.py`) → `{member_count, total_visits, today_visits}`(가입자 총수 + 방문 누적·오늘 KST). `POST /api/visit`(앱 로드 1회 기록·누적 +1·오늘 날짜경계 리셋). **공개·인증 불요·집계만(PII 없음)·항상 200 graceful**. 백엔드 `infra/site_stats.py`(`SiteStat` 싱글턴 행·`import_models` 등록). 톱바 헤드라인이 표시.
- **잔고(포트폴리오, UX 개편)** — `GET /api/balance`(`api/balance.py`) → `{holdings, summary, partial_failure}`. `api.detail._build_kis_client`로 클라이언트 조립 후 `collectors.kis.balance.inquire_balance(client, cano, prdt)` 호출. **조회 전용(주문/매매 API 없음)**, 현재가(prpr) 포함이라 **캐시 미경유**(원칙1). **각 holding 에 `spark`(관심종목과 동일 미니차트)를 라우트가 후처리로 얹는다** — `watchlist.service.fetch_sparks_parallel`(공용) 재사용, spark 실패는 `spark=None`·`partial_failure` 미오염(선택적 시각화·독립). 계정은 `infra.config.kis_account()`(=`KIS_ACNT_NO`→미설정 시 `KIS_ACCOUNT_NO` 폴백, `KIS_ACNT_PRDT_CD_STK`→기본 `"01"`). KIS 실패는 graceful — 항상 **200** + `holdings=null·summary=null·partial_failure:['balance']`(프론트가 "일시 조회 불가" 안내). GET 전용. 테스트는 StubClient+fixture로 shape·partial_failure 고정(`tests/unit/api/test_balance_route.py`).
- `allow_methods`에 `GET`·`POST`·**`DELETE`·`PATCH`** 허용(챗봇 POST + 워치리스트 CRUD 때문에 DELETE/PATCH 추가).
- **`api/deps.py` = 라우트 공용 SSOT**(IMP-02/06): ① `assert_valid_ticker`(모든 `{ticker}` 라우트 진입부 400 — 불량 코드가 KIS/OpenAI/저장 트리거 전 차단), ② 국면 4지표 매핑 `_REGIME_INPUT_MAP`·`map_engine_input`(단일 출처), `build_judgement`(종목/워치리스트/리포트가 소비). `macro_regime`·챗봇의 `live_judgement`는 `api/main.py`에 남기되(라우트 테스트가 `main.collect_macro_indicators`/`judge_regime`을 경계 patch) 매핑만 `deps`를 쓴다 — 매핑을 한쪽만 고쳐 국면 입력이 갈리는 부채 제거. `deps`는 `api.*`를 import하지 않아 사이클 없음.

## 유저베이스 — 인증·DB·대화기록 (Phase 2~5)
- **DB = SQLAlchemy**(`infra/db.py`): `DATABASE_URL` env(로컬 기본 `sqlite:///.cache/app.db`, 프로덕션 **GCP Cloud SQL for PostgreSQL** `postgresql+psycopg://…`). `init_db()`(startup)가 `create_all`(Phase마다 신규 테이블 추가라 alter 불요). 모델 등록 단일 지점 `import_models()`. 요청 스코프 세션 `get_db` 의존성(테스트는 `dependency_overrides[get_db]`로 인메모리 주입).
- **인증** — `api/auth.py`(signup/login/me), `auth/deps.get_current_user`(Bearer JWT→User, 401). 상세 `auth/CLAUDE.md`.
- **관심종목은 이제 인증 스코프**(Phase 3): `api/watchlist.py`가 `DEFAULT_USER_ID`/query·body user_id 제거 → `get_current_user.id` 주입(모든 CRUD 401 without token). store 는 `SqlWatchlistStore`(기존 `WatchlistStore` Protocol 구현·`(user_id,ticker)` 유니크) — service·프론트 계약 불변. 유저 격리(A 종목 B 안 보임).
- **대화기록**(Phase 4) — `api/conversations.py`: `GET/POST /api/conversations`·`GET /api/conversations/{id}/messages`·`DELETE`(모두 유저 스코프·소유권 404). 챗 라우트(`/api/chat`·`/api/chat/stream`)는 **인증 필수** + `session_id=conversation.id`: 세션 DB hydrate(재접속·전환·재시작 컨텍스트 복원, best-effort) + 턴 write-through. `Conversation`/`ChatMessage` 모델·`HistoryStore`. 스트림은 토큰 누적 후 저장(db Session 은 스트리밍 종료까지 생존).
- **유저별 KIS 자격증명**(공유키 폐기) — `api/kis_credentials.py`: `POST /api/me/kis-credentials`(인증, 실제 KIS 토큰 발급으로 **검증 후** 암호화 저장·실패 400·키값 미노출)·`GET`(마스킹 상태만)·`DELETE`(삭제→공유 fallback). **KIS 라우트는 옵션 인증**: `balance`·`detail bundle`·`report` 는 `get_current_user_optional`+`resolve_kis_client(user,db)`로 **공개 유지하되 로그인+등록 시 본인 KIS 키**(아니면 `__shared__`→env fallback). `watchlist`(이미 인증)·`chat/view_context`(잔고/관심/종목 스냅샷)도 유저 키 스레딩(`/api/chat/context`는 user/db 전달). 자격증명 없음·KIS 실패는 graceful(항상 200). 상세 `auth/CLAUDE.md`·`collectors/CLAUDE.md`.
- **잔고는 이제 유저별**(등록 시 본인 계좌, 미등록/비로그인은 공유 데모 계좌 `__shared__`) — env `KIS_ACNT_NO`는 로컬 fallback·프로덕션 시드 원천.

## 관리자 API + 챗 질문 한도 enforcement
- **`api/admin.py`**(전 라우트 `get_admin_user` 게이트·비관리자 403): `GET /api/admin/users`(유저 목록+이용 통계: is_admin·daily_limit·used_today·remaining·total_questions·created_at, **비밀번호 해시·KIS 원문 미노출**)·`PATCH /api/admin/users/{id}`(is_admin·daily_limit **부분 갱신** `model_fields_set`, 음수 422, 없음 404, **자기 자신 관리자해제 400**)·`POST …/{id}/reset-usage`(오늘 사용량 0, 누적 통계 보존)·`DELETE …/{id}`(유저 + 스코프 데이터[관심종목·대화(+메시지 cascade)·KIS 자격증명] 정리, **자기 자신 삭제 400**). 매매·자격증명 원문 무관(조회/제어만). `api/main.py`가 kis_credentials 라우터 뒤에 include.
- **챗 질문 한도 enforcement**(`api/chat.py`, `auth.usage`): `/api/chat`·`/api/chat/stream` **진입부에서 `is_over_limit(user)` 선차단** → LLM 미호출, 안내문만 반환/스트림(비용 절감). 성공 턴마다 `consume(user, db)`(일별 리셋·누적·commit). 관리자 무제한. 라우트 테스트 픽스처는 한도 여유 유저 + `commit` 가능한 더미 db 주입(대화기록 helper 는 no-op).
