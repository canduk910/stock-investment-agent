# UI 디자인 시스템

이 프로젝트의 모든 화면(매크로 대시보드 · 종목 리포트 · 챗봇 · 워치리스트 팝업)은 하나의 톤을 공유한다. 목적: 전문적이고 신뢰감 있는 금융 도구 느낌 + 색으로 판단을 유도하지 않는 절제된 표현.

## 1. 색 팔레트 — 흰색 / 회색 / 파랑 / 남색 / 검정 (+ 강조 주황 · 위험 빨강)

**이 5개 계열 + 주황(강조) + 빨강(위험)만 쓴다.** 초록·황색 등 그 외 색은 쓰지 않는다.

| 계열 | 용도 |
|------|------|
| **흰색** | 카드·패널 표면, 배경(아주 옅은 회색과 함께) |
| **회색** | 테두리, 보조/흐린 텍스트, **보합(중립) 상태**, 보조 표면 |
| **파랑** | 기본 액센트, 인터랙션(버튼 hover·링크), 정보 배너, **하락·손실 방향** |
| **빨강**(`--c-up`) | **상승·수익 방향**(한국 관습, 전 화면). 위험 경보(`--c-danger`)와 형태로 구분. |
| **남색** | 제목·주요 텍스트, 정보 배너, 배지, 히어로/유저버블 존 |
| **검정** | 카드 값 숫자 등 |
| **주황**(`--c-emph`) | **강조** — 국면명·권장 현금비중·목표가 상태·진입 검토·알림·확인 CTA. 가격 방향엔 금지. |
| **빨강 경보**(`--c-danger`) | **위험** — 오직 손실경고 배너·VIX 패닉 칩. 채움형 배너/칩 + ⚠ 로 상승 빨강과 구분. |

### 가격/손익 방향 = 빨강(상승)/파랑(하락) — 사용자 확정, 전 화면
한국 시장 관습(상승=빨강)을 **모든 주식 수치**에 적용한다(등락률·손익·PER 추이 등). 이제 캔들차트만의 예외가 아니다.
- **상승·수익 → 빨강(`--c-up` #e5322d)** / **하락·손실 → 파랑(`--c-down` #2563eb)** / **보합 → 회색(`--c-flat`)**.
- **색만으로 구분 금지** — 항상 ▲▼─ 글리프를 병기한다(접근성·색약 대응).

### 난색(warm) 두 색의 역할 분리 — 절대 섞지 않는다
상승 빨강(방향)과 위험 빨강(경보)이 같은 계열이라 **형태로 구분**한다. 남발하면 신호가 무뎌진다.
- **주황 = 강조**: 국면명·현금비중·목표가 도달/근접·진입 검토가능·알림 배너·확인 CTA. "여기를 보라"(위험 아님).
- **빨강 경보(`--c-danger`) = 위험**: 손실경고 배너·VIX 패닉 칩만. **항상 채움형 배너/칩 + ⚠ 글리프**로 방향 빨강(텍스트+▲)과 형태 구분. 강도 최상, 빈도 최소.
- 강조·경보에 **가격 방향색을 쓰지 않는다**(방향은 빨/파, 강조는 주황, 경보는 채움 빨강).
  - **캔들차트(KLineChartPanel)**: 전용 토큰 `--c-chart-up`(빨강)·`--c-chart-down`(파랑)을 쓴다 — 값은 `--c-up/--c-down`과 같아졌지만 klinecharts 주입 경로(`lib/theme.js::readChartPalette`) 보존을 위해 토큰은 분리 유지. 지표선(MA/RSI)·오버레이(52주·현재가선)는 남색·회색으로 캔들색과 구분한다.

### 상태 표현은 팔레트 안에서 (중요)
- **상승·수익** → 빨강(`--c-up`) / **하락·손실** → 파랑(`--c-down`) / **보합·중립** → 회색(`--c-flat`)
- **정보** → 남색(`--c-info`) / **강조** → 주황(`--c-emph`) / **손실 위험·경보** → 빨강 경보(`--c-danger`, 채움형)

## 2. 디자인 토큰이 유일한 색 출처

색은 `frontend/src/theme.css`의 CSS 변수(토큰)에만 정의한다. 컴포넌트 CSS·JSX는 **하드코딩 hex를 쓰지 않고 `var(--c-...)`로만 참조**한다. 색을 바꾸려면 토큰 한 곳만 바꾼다(임계값 3중 일관성과 같은 SSOT 원칙의 UI판).

```css
/* 나쁨: 하드코딩 */
.card__value { color: #0b111c; }
/* 좋음: 토큰 참조 */
.card__value { color: var(--c-black); }
```

새 색이 필요하다고 느끼면 먼저 기존 토큰으로 표현 가능한지 본다. 정말 필요하면 `theme.css`에 토큰으로 추가하되 5개 계열 안에서 만든다.

### 토큰 목록 (theme.css)
- 표면/배경: `--c-white`, `--c-bg`, `--c-surface`, `--c-surface-2`
- 테두리: `--c-border`, `--c-border-strong`
- 텍스트: `--c-black`, `--c-navy`, `--c-text`, `--c-text-secondary`, `--c-text-muted`
- 액센트: `--c-blue`, `--c-blue-strong`, `--c-blue-soft`
- 상태(방향): `--c-up`(빨강=상승), `--c-down`(파랑=하락), `--c-flat`(회색=보합) + 소프트 배경 `--c-up-soft`·`--c-down-soft`·`--c-flat-soft`·`--c-up-onnavy`
- 강조/위험/정보: `--c-info`(남색), **`--c-emph*`(주황 — 강조)** + `--c-emph-onnavy`, **`--c-danger*`(빨강 경보 — 위험)**
- 브랜드/셸: `--c-brand`(워드마크 파랑), `--c-navy-deep`(히어로/유저버블), 보조 표면 `--c-surface-3`·`--c-border-soft`·`--c-hairline`·`--c-text-faint`
- 형태: `--radius`(14px), `--radius-sm`, `--radius-lg`(섹션 카드), `--radius-xl`(셸 카드), `--radius-pill`, `--shadow-card`, `--shadow-raise`

## 3. 컴포넌트 규칙

- **테마**: 라이트(흰색 배경) 기준. 카드는 흰색 표면 + 옅은 회색 테두리 + 미세한 그림자로 깊이를 준다.
- **텍스트 위계**: 제목=남색, 주요 값=검정, 본문=`--c-text`, 보조=회색(`--c-text-secondary`), 메타=`--c-text-muted`.
- **카드**: `--radius`(14px, 셸 카드는 `--radius-xl` 18px) 라운드, `--shadow-card`, hover 시 `--shadow-raise` + 테두리 강조. 실패(부분 실패) 카드는 dashed 테두리 + 보조 표면색으로 조용히 구분.
- **버튼/인터랙션**: 기본은 흰 표면 + 회색 테두리, hover에서 파랑으로 전환.
- **배지/배너**: 배지는 남색 배경 + 흰 글씨. 경고/부분실패 배너는 옅은 파랑 배경(`--c-blue-soft`) + 파랑 텍스트. 하드 에러 배너는 남색 테두리 강조.
- **간격/타이포**: 값 숫자는 크고 굵게(28px/700), 라벨·메타는 작고 절제. 과한 장식 지양 — 5주 과제 범위에서 스냅샷 남발·픽셀 검증은 하지 않고 톤 일관성만 지킨다.

## 4. 접근성 최소선

- 흰 배경 위 본문 텍스트는 `--c-text`(#1e2a44) 이상으로 충분한 대비 확보. 흐린 메타(`--c-text-muted`)는 보조 정보에만.
- 색만으로 상태를 구분하지 않는다 — 상승/하락은 화살표(▲▼) 글리프를 함께 쓰고, 부분 실패는 텍스트 안내("일시 조회 불가")를 함께 둔다.

## 5. 왜 이렇게 하는가

토큰 SSOT + 제한된 팔레트는 화면이 여러 개(대시보드·리포트·팝업·챗봇)로 늘어나도 톤이 흩어지지 않게 한다. 새 화면을 만들 때 색을 새로 고르지 말고 이 토큰을 조합하면, 전체가 한 제품처럼 보인다.
