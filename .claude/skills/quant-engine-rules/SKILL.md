# 정량 엔진 구현 규칙

원본 스펙은 `invest_develop_PLAN.md` §4·§6.1·§6.5a. 이 스킬은 구현 시 지켜야 할 규칙과 함정을 요약한다. 충돌 시 플랜 문서가 우선한다.

## 1. 절대 원칙

- **LLM 미개입**: 이 계층의 모든 산출(국면, 현금비중, 신뢰도, CAGR, 판정 라벨)은 결정적 코드가 계산한다. LLM 호출 코드가 이 모듈에 들어가면 안 된다.
- **3중 일관성**: 임계값은 코드 상수 1곳에 정의한다. 같은 숫자를 두 곳에 쓰는 순간 언젠가 어긋난다.
  - 상수 목록: `THRESHOLDS`(지표별 구간), `CASH_RATIO`, `REGIME_PARAMS`, `VIX_PANIC = 35`, `VALUATION_BAND = 0.10`(±10% 판정 라벨)
  - 프롬프트 기준표는 llm-engineer가 이 상수에서 자동 생성한다 → **상수의 키 이름·구조를 바꾸면 llm-engineer에게 알려야 한다**
- **순수 함수**: 입력 dict → 출력 dict. API 호출·DB 접근은 데이터 계층의 일이다.

## 2. 매크로 엔진 — 판정 순서 (순서가 중요하다)

```
1. VIX 패닉 오버라이드 검사 (data["vix"] > VIX_PANIC → 즉시 수축, 투표·하이스테리시스 무시)
2. 가중 투표 (score_indicators)
3. [P2] 하이스테리시스 (1·2위 격차 ≤1이고 2위가 직전 국면이면 직전 국면 유지)
4. 신뢰도 = 1위-2위 격차 기반 (gap≥3 high / gap≥2 medium / 그 외 low)
```

오버라이드가 투표보다 먼저다 — 시장 패닉엔 평균(투표)이 극단 신호를 희석하기 때문. 하이스테리시스는 오버라이드에 적용하지 않는다.

### 투표 규칙 (플랜 §4 표 기준)

- 신용·금리 그룹(금리차, HY스프레드) = 가중치 2 / 나머지 = 1 (공포탐욕지수가 VIX 등을 이미 포함하는 정보 중복 보정)
- 1차 구현은 정수 투표로 완성 → [P2] 분수 점수제(합계 1.0 배분)로 전환. 분수 전환 시에도 `THRESHOLDS` 경계는 유지하고 "정상" 구간만 인접 국면에 분할 배분
- 반환 dict에 `votes`, `key_drivers`(기여 지표), `raw_data`, `override`를 반드시 포함 — LLM 설명의 재료다

### 부분 실패 처리

지표가 누락되면(수집 실패) 해당 지표를 투표에서 제외하고 결과에 `missing_indicators: [...]`를 명시한다. 임의 기본값(0, 평균 등)으로 채우면 판정이 조용히 왜곡된다.

## 3. REGIME_PARAMS — 국면별 실행 파라미터

```python
REGIME_PARAMS = {
    # 관점 통일 주의: 아래는 "현금비중(%)" 기준. 플랜 §3의 표기와 관점을 하나로 정하고 한 곳에서만 정의할 것.
    "회복": {"cash": 20, "single_cap": 5, "per_max": 20, "pbr_max": 2.0},
    "확장": {"cash": 40, "single_cap": 4, "per_max": 15, "pbr_max": 1.5},
    "과열": {"cash": 60, "single_cap": 3, "per_max": 12, "pbr_max": 1.2},
    "수축": {"cash": 80, "single_cap": 0, "per_max": None, "pbr_max": None},
}
```

- `single_cap=0`은 "신규 진입 제안 자체를 하지 않음" — 모듈 3 진입 신호의 게이트
- **함정**: 플랜 §3의 `CASH_RATIO`(회복 80/확장 60/과열 40/수축 20)는 "주식비중" 관점 표기라서 위 표와 숫자가 반대로 보인다. 구현 전에 **"현금비중" 관점으로 통일**하고 한 상수만 남긴다. 두 상수가 공존하면 3중 일관성 위반이다.

## 4. stock_summary — 종목 정량 요약

```python
{
  "rev_cagr": ..., "op_cagr": ...,        # 확보한 재무 연수 기준 CAGR
  "current_per": ..., "avg_per": ...,      # avg = 조회 기간 평균 (자기 자신과의 비교)
  "per_vs_avg": ...,                       # (현재-평균)/평균 %, 음수 = 평균보다 저렴
  "valuation_label": ...,                  # ±10%: 저평가 / 적정 / 고평가 — 코드가 붙인다
  "rsi": ..., "ma20_gap_pct": ..., "pos_52w_pct": ...,
}
```

- 핵심 아이디어: "PER 15가 싼가"는 절대 기준이 없지만 "자기 5년 평균 대비 −18%"는 근거가 명확하다. 판정 라벨은 코드가 붙이고 LLM은 바꿀 수 없다.
- CAGR 계산 함정: 시작값이 0 이하(적자→흑자)면 CAGR이 정의되지 않는다 — `None` 반환 + 이유 기록. 억지로 계산하지 않는다.
- RSI·이동평균은 KIS 일봉(`inquire_daily_itemchartprice`) 입력으로 계산. 데이터 부족(신규 상장 등) 시 해당 필드 `None`.

## 5. 단위 테스트 필수 케이스

경계값에서 판정이 갈리므로 경계 자체를 테스트한다. **TDD 순서 준수(`tdd-workflow` 스킬): 아래 목록을 구현 전에 테스트 파일로 먼저 옮기고 실패를 확인한다** — 이 목록이 곧 Red 단계다:

- 각 임계값 경계: `yield_spread` = 0 / 0.5, `vix` = 14 / 28, `fear_greed` = 25 / 75, `buffett` = 150 — 경계값이 어느 구간에 속하는지(초과 vs 이상)를 명시적으로 고정
- **VIX 오버라이드**: `vix = 35`(발동 안 함, `>` 조건)와 `vix = 35.1`(발동) — 다른 지표가 전부 과열이어도 수축이 나오는지
- 투표 동점: `max()`는 삽입 순서 첫 키를 반환한다 — 동점 규칙(보수적으로 더 방어적인 국면 선택 권장)을 정하고 테스트로 고정
- [P2] 하이스테리시스: 격차 1 + 직전 국면 2위 → 유지 / 격차 2 → 전환
- stock_summary: 적자 전환 CAGR `None`, ±10% 경계 라벨
- 테스트가 실패하면 로직을 고친다 — 임계값 상수를 테스트에 맞춰 바꾸지 않는다
